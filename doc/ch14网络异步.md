## 异步的本质
- 多个玩家中 游戏状态不同
### 举例说明
```
玩家1的电脑
    玩家1的剑圣
        生命 20  -> 30              
    玩家1的单位2

    玩家2的圣骑士                                        网易服务器
        生命 30

玩家2的电脑
    玩家1的剑圣
        生命 20

    玩家2的圣骑士
        生命30
```
1. 多个玩家电脑中状态同步方式 服务器定时检测玩家的状态
2. 如果多个电脑之间状态不同 服务器就会发送异步错误警告到客户端
3. 导致状态不同的原因     玩家本地操作修改状态 没有发送修改信息到服务器
    - ui操作
    - 代码操作


## 异步的解决办法 数据同步
DzSyncData
DzTriggerRegisterSyncData

DzGetTriggerSyncPlayer
DzGetTriggerSyncData
- 同步数据最好参考的框架实现

```lua
-- 接收到的状态
hsync.receive("hzg", function(syncData)
    echo(syncData.triggerData[1] .. syncData.triggerData[2] .. "人类")
end)

-- 发送本地状态
local a = 0
htime.setInterval(2, function(curTimer)
    hsync.send("hzg", { "hunzsig", "是个" .. a .. "级" })
    a = a + 1
end)
```
